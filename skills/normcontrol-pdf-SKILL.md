# Скилл: Нормоконтроль PDF-документации

## Цель
Автоматическая проверка проектной и рабочей документации (PDF) на соответствие:
- **ГОСТ Р 21.101-2026** — основные требования к ПД и РД
- **ГОСТ 21.110-2013** — спецификация оборудования, изделий и материалов
- Требованиям к ЦИМ (атрибутивный состав, матрица коллизий)

На выходе — **Excel-отчёт** с замечаниями по каждому листу.

---

## Шаг 1 — Определить задачу

Уточни у пользователя:

1. **Что проверяем?** — штамп, таблицы, или всё вместе
2. **Источник** — папка с PDF / один файл / уже загружен
3. **Тип PDF** — текстовый (из Revit/AutoCAD) или скан
4. **Желаемый результат** — Excel, отчёт PDF, интерактивный дашборд

> Если пользователь говорит «проверь всё» — проверяем штамп + все таблицы.

## Шаг 2 — Диагностика PDF (обязательно перед парсингом)

Перед написанием парсера **всегда** запускай диагностику:

```bash
python normcontrol/diagnose_pdf.py --file лист.pdf --debug
```

**Что смотрим в выводе:**

| Секция диагностики | Что означает | Действие |
|---|---|---|
| Раздел 2: «Найдено таблиц: N» | extract_tables работает | Парсим через extract_tables() |
| Раздел 6: «Линий нет» | Таблицы текстовые (пробелы) | Парсим через extract_words() |
| Раздел 1: текст не извлечён | Скан | Нужен OCR (Tesseract) или Claude Vision API |

Полный скрипт диагностики → `references/diagnose_pdf.py`

## Шаг 3 — Проверка штампа (ГОСТ Р 21.101-2026)

### Зона штампа
Стандартный Revit/AutoCAD-штамп — **нижние 25–30% страницы**, правый угол.

```python
STAMP_ZONE_RATIO = 0.30  # нижние 30%
```

### Обязательные поля штампа

| Поле | Ключевые слова | Критичность |
|---|---|---|
| Марка раздела | ОВ, ВК, АР, КР, ЭОМ… | ЗАМЕЧАНИЕ |
| Обозначение документа | паттерн «ХХ-МА-001» | ЗАМЕЧАНИЕ |
| Стадия | П, Р, РД, ИД, ПД, РП, ЭП | ЗАМЕЧАНИЕ |
| Лист № | «Лист» | ЗАМЕЧАНИЕ |
| Листов | «Листов» | ЗАМЕЧАНИЕ |
| Масштаб | «Масштаб», «1:N», «НУ» | ЗАМЕЧАНИЕ |
| Подпись «Разработал» | «Разработал», «Разраб» | ЗАМЕЧАНИЕ |
| Подпись «Проверил» | «Проверил», «Пров» | ЗАМЕЧАНИЕ |
| Подпись «Н/контроль» | «Н.контроль», «Н/контроль» | только для стадий Р, РД, ИД |
| Подпись «ГИП» | «ГИП», «Г.И.П» | на титульных листах |

**Допустимые марки разделов:**
АР, КР, КЖ, КМ, КД, ОВ, ВК, ВКН, НВК, ЭОМ, ЭС, ЭН, СС, АТХ, ТХ, ПБ, ГП, ПТ, ОДИ, ИОС1–ИОС9, АПТ, ОТ

**Допустимые стадии:** П, Р, РД, ИД, ПД, РП, ЭП, ПОС, ППР

## Шаг 4 — Проверка таблиц

### 4.1 Ведомость чертежей (ГОСТ Р 21.101-2026, Форма 1)

Обязательные графы:

| Графа | Проверка |
|---|---|
| **Лист** | Не пустая, арабские цифры |
| **Наименование** | Совпадает с наименованием в штампе листа |
| **Примечание** | Необязательна, но при изменениях — заполнена |

### 4.2 Спецификация оборудования (ГОСТ 21.110-2013, Форма 1)

**9 граф** — полная форма:

| Графа | Обязательность | Исключения |
|---|---|---|
| **Поз.** | Обязательна | Цифровое или буквенно-цифровое обозначение |
| **Наименование и техническая характеристика** | Обязательна | — |
| **Тип, марка, обозначение документа** | Обязательна | Не заполняют для изделий инд. изготовления |
| **Код продукции** | Необязательна | Не заполняют если нет в документации |
| **Поставщик** | Необязательна | — |
| **Ед. измерения** | Обязательна | — |
| **Кол.** | Обязательна | Допускается не заполнять для м²/м³ |
| **Масса 1 ед., кг** | Условно | Не заполняют для изделий ≤25 кг без подъёмных средств |
| **Примечание** | Необязательна | — |

**Логика нумерации позиций (п. 4.5 и 4.7 ГОСТ 21.110-2013):**
- Позиции нумеруются в пределах раздела (как правило)
- Порядковый номер записи указывается перед наименованием
- При делении на разделы — нумерация может начинаться заново в каждом разделе
- Позиционное обозначение — цифровое или буквенно-цифровое

### 4.3 Упрощённая спецификация (ГОСТ Р 21.101-2026, Форма 7)

6 граф — базовая форма:

| Графа | Обязательность |
|---|---|
| Поз. | Обязательна |
| Обозначение | Обязательна |
| Наименование | Обязательна |
| Кол. | Обязательна* |
| Масса ед., кг | Обязательна* |
| Примечание | Необязательна |

### 4.4 Экспликация помещений

Обязательные поля (из ГОСТ Р 21.101-2026 + требования к ЦИМ):

| Поле | Проверка |
|---|---|
| Номер помещения | Уникальный, не пустой |
| Назначение | Не пустое |
| Площадь | Числовое значение > 0, по СП 118.13330.2012 |
| Высота (опционально) | Числовое значение |

### 4.5 Таблица изменений (угловой штамп)

6 граф по ГОСТ Р 21.101-2026:

| Графа | Проверка |
|---|---|
| **Изм.** | Порядковый номер, не пустой |
| **Кол. уч.** | Прочерк при полной замене листа |
| **Лист** | «Зам.» / «Нов.» / «Все» — по правилу |
| **№ док.** | Не пустой |
| **Подп.** | Обязательна |
| **Дата** | Обязательна |

## Шаг 6 — Требования к ЦИМ

При проверке BIM/ЦИМ-документации дополнительно проверяем:

**Атрибутивный состав:**
- Значения атрибутов ЦИМ должны совпадать с данными в PDF-таблицах
- Для IFC: атрибуты сгруппированы в наборы свойств, именованы по стандарту

**Помещения (IfcSpace):**
- Номер, Назначение, Площадь (по СП 118), Высота, Периметр
- Дополнительно: Вместимость, доступность МГН, класс пожарной опасности

**Обязательные сопроводительные документы ЦИМ:**
- Матрица коллизий (xlsx, ods или pdf-a)
- Отчёт по коллизиям

## Веб-приложение (рекомендуется)

Самый удобный способ — виджет прямо в чате Claude. Не требует установки ничего.

**Возможности:**

- Поддерживает многостраничные PDF (весь том одним файлом)

## Типичные проблемы и решения

| Проблема | Симптом | Решение |
|---|---|---|
| Штамп не читается | Все поля «—» | Запусти `--debug`, проверь STAMP_ZONE_RATIO |
| Таблицы не извлекаются | extract_tables() = [] | Таблицы текстовые — используй extract_words() |
| Скан PDF | Текст = пустой | Нужен OCR: Tesseract или Claude Vision API |
| Нестандартный штамп | Поля не распознаются | Добавь ключевые слова в parse_stamp() под конкретный шаблон |
| Кириллица не читается | Мусор вместо текста | Проверь кодировку PDF, попробуй другой PDF-ридер |
