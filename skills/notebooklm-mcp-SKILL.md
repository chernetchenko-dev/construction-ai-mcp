# NotebookLM MCP

## Доступные инструменты

| Инструмент | Назначение |
|---|---|
| `notebook_list` | Список всех нотбуков |
| `notebook_get` | Детали нотбука + источники |
| `notebook_describe` | AI-резюме нотбука + предлагаемые темы |
| `notebook_query` | Вопрос по существующим источникам |
| `source_list_drive` | Список источников нотбука |
| `source_describe` | AI-резюме конкретного источника |
| `source_get_content` | Сырой текст источника без AI |
| `research_start` | Поиск НОВЫХ источников в вебе или Drive |
| `research_status` | Статус запущенного ресёрча |
| `source_sync_drive` | Синхронизация источников из Drive |

---

## Правила выбора инструмента

### notebook_query — вопросы по существующим документам
Используй когда пользователь хочет найти информацию в уже загруженных источниках.

```
notebook_query(
  notebook_id = "...",
  query = "конкретный вопрос",
  source_ids = ["id1", "id2"],  # опционально — фильтр по источникам
  conversation_id = "...",       # опционально — для follow-up вопросов
)
```

**ВАЖНО:** `notebook_query` — только для существующих источников. Для поиска новых → `research_start`.

### research_start — найти новые источники
Используй когда нужно найти информацию в интернете или Google Drive и добавить в нотбук.

```
research_start(
  query = "тема поиска",
  source = "web",        # web | drive
  mode = "fast",         # fast (~30с, ~10 источников) | deep (~5мин, ~40 источников)
  notebook_id = "...",   # опционально — добавить в существующий
  title = "...",         # опционально — создать новый с этим именем
)
```

После запуска — `research_status` для проверки, затем `research_import` для добавления источников.

## Алгоритм работы

### Шаг 1 — Определить нотбук
Если нотбук не указан явно → `notebook_list` → выбрать по названию.

### Шаг 2 — Выбрать инструмент
```
Вопрос по документам?        → notebook_query
Найти новые источники?       → research_start
Список источников?           → source_list_drive
Резюме нотбука?              → notebook_describe
Резюме источника?            → source_describe
Сырой текст источника?       → source_get_content
```

### Шаг 3 — Уточнить запрос
- Передавай `conversation_id` из предыдущего ответа для follow-up вопросов
- Передавай `source_ids` если известно в каком файле лежит информация — быстрее и точнее
- Формулируй запрос конкретно: указывай типоразмер, артикул, стандарт

## Типовые сценарии

### Спецификация из каталога производителя
```
1. notebook_list → найти нотбук "Каталоги"
2. source_list_drive → посмотреть какие PDF есть
3. notebook_query → запросить артикул + масса + характеристики
4. Если нужен конкретный PDF → передать source_ids
```

Пример запроса: `"Листовой лоток 300х100 артикул, масса 1м, горизонтальный поворот 90°"`

### Вопрос по нормативному документу
```
1. notebook_list → найти нужный нотбук (ГОСТ, СП, ФОП)
2. notebook_query → конкретный вопрос со ссылкой на раздел
3. Передавать conversation_id для уточняющих вопросов
```

### Глубокий ресёрч по теме
```
1. research_start(query="тема", mode="deep", title="Новый нотбук")
2. research_status → дождаться завершения
3. research_import → добавить найденные источники
4. notebook_query → начать задавать вопросы
```

### База знаний проекта
```
1. notebook_list → найти нотбук проекта (ПИР, Г5-ОВ и т.д.)
2. notebook_describe → получить общее резюме
3. notebook_query → конкретный вопрос по теме
```

## Ошибки авторизации

Если инструмент возвращает `Authentication expired` или `RPC Error 16`:

**macOS:**
```bash
~/.cache/uv/archive-v0/*/bin/notebooklm-mcp-auth
# или если настроен алиас:
nblm-auth
```

**После авторизации — обязательно перезапустить Claude Desktop (Cmd+Q).**

Windows: запустить `notebooklm-mcp-auth.exe` из папки uv cache, перезапустить Claude Desktop.

## Ограничения

- PDF с таблицами-картинками (сканы) не индексируются — нужен текстовый слой
- Файлы >50 МБ загружаются со статусом `unknown` и не индексируются — разбивать на части
- Сессия авторизации живёт несколько дней/недель, после чего нужна повторная авторизация
